1. item1: view c++ as a federation of language

可以从 c , object-Oriented C++, Template C++, The STL， 这四个方面来理解C++这个语言， 分别对应四种不同的使用目的。

2. item2: prefer consts, enums, and inline to #defines

这条建议也可以理解成：更倾向于编译器，而非预处理器，这是因为#defines 可能不被包含在语言内。例如：

#define asb_ratio 1.45

asb_ratio 这个符号可能永远不会被编译器观测到，在送到编译器里前，这个可能被预处理器删除。因此，这个变量可能不会进入符号表。解决方法是，使用const来代替macro:

const double asb_ratio=1.45

​ 使用 #define 的另外一个目的是：定义宏，使它表现地像函数，而没有函数调用的开销。但是这个用法需要注意的东西很多，例如：

​ #define CALL_WITH_MAX(a, b) f((a) > (b) ? (a) : (b))

int a = 5, b = 0;
CALL_WITH_MAX(++a, b); // a is incremented twice
CALL_WITH_MAX(++a, b+10); // a is incremented once

根据形参的不同， 对 a 有不同的操作。const ,enums, 和inlines 的引入，预处理器的需要有所下降，但没有完全消除，比如#ifdef/#ifndef 依然在控制编译处发挥这作用。以上宏定义可以使用模板进行更改：

template // because we don’t
inline void callWithMax(const T& a, const T& b) // know what T is, we
{ // pass by reference-to

​ f(a > b ? a : b); // const — see Item 20
}

3. item 3：Use const whenever possible

​ const 关键字有多变灵活的使用目的。在类外，可以在 namspace 或 global 范围内定义常量，或者在文件，函数，块域修饰使用static 修饰的对象。在类内，可以对static 或者non-static 的数据成员进行修饰。对于指针，可以指定指针本身是否是const, 指针指向的对象是否是const.

Antman

发表于 2021-05-31 更新于 2021-06-01

发现：在多租户环境下，GPU利用率低。—> 原因：①：大多数DL训练任务，在其训练过程中，不能充分地利用GPU资源。 主要体现在：1）训练DL 模型需要混合地计算需求，其中一些很难并行化。2）当

并行化时，90%的时间，用在网络上。 ②：集群调度中的预留策略导致大量的GPU空闲时间，因为DL Job不能使用部分的资源。比如，SGD是同步的算法，需要所有资源在线，以用于全局调度。导致调度器需要预留部分资源，直到任务需求满足要求为止。

综上所述，可以发现如果在GPU集群上，运行多个Job，能够提高GPU的利用率–>容纳更多的Job。然而这种方法，鲜有在生产环境中使用，这是因为除了提高GPU的利用率外，还需要保证有资源需求的Job的性能。将多个Job同时在一个GPU上运行，可能会有冲突，导致性能的显著下降。

本文提出AntMan, 在保证公平性和resource-guaranteed job 的性能的同时，提高GPU集群的利用率—>通过合作式的资源扩展来最小化 Job的interference.–> 该机制通过微改DL框架和cluster scheduler，来动态分配刚好的GPU内存和计算单元(这两个概念如何理解，需要学习GPU的知识)，任何的GPU空闲资源(内存和计算)，可以通过over-subscrpition jobs 进行利用（这个技术是超预定技术，时用户的需求暂时超过cluster的资源量—>提高用户的满意度）。

AntMan 相比于先前的调度器，提升了Job completion Time 和GPU 的内存和计算利用率。

AntMan 设计由以下三个部分组成：①：DL frameworks的extension ② 合作式的使用这些primitive进行调度。③：集群中的调度策略 （ 点 从下到大）
